<threat_model>
- **Threat:** Cross-user access caused by child rows not being scoped through the parent comic. **Mitigation:** Preserve/extend comic ownership constraints and require later service queries to resolve owned comics before child mutations.
- **Threat:** Data integrity loss from unstructured JSON-only persistence. **Mitigation:** Add first-class fields and tests for Phase 5 contract data.
- **Threat:** Broken migrations in local/prod environments. **Mitigation:** Add Alembic migration tests and keep changes inside backend schema files.
</threat_model>
