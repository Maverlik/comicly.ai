<threat_model>
- **Threat:** Browser-supplied owner ids or nested ids could bypass ownership. **Mitigation:** Do not accept owner ids from requests; route uses current user and service-owned comic lookup.
- **Threat:** API leaks existence of another user's comic. **Mitigation:** Return the same not-found response for missing and foreign-owned comics.
- **Threat:** Future frontend depends on unstable response shapes. **Mitigation:** Use explicit Pydantic request/response models and cover contracts in tests.
</threat_model>
