<threat_model>
- **Threat:** IDOR access to another user's comics, scenes, or pages. **Mitigation:** Service methods accept `user_id` and resolve/mutate child data only through an owned comic query.
- **Threat:** Accidental data loss from hard deletes or list overwrite bugs. **Mitigation:** Use soft archive for comics and test scene/page replacement behavior.
- **Threat:** Invalid draft data causing unusable reloads. **Mitigation:** Validate required metadata/order/page fields before mutation and test full detail reconstruction.
</threat_model>
