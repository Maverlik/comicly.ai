Connect the current creator state to private comic persistence without adding comic history UI.

Output: current-comic creation/update flow, hybrid save triggers, and compact save status.

<threat_model>
- **Threat:** Frontend sends generation without a real owned `comic_id`. **Mitigation:** Ensure current comic exists before generation and store the returned id in memory.
- **Threat:** Phase drifts into history/reopen UI. **Mitigation:** Use only current-comic APIs; do not render old comic list/detail navigation.
- **Threat:** Client trusts owner or balance values. **Mitigation:** Only backend response ids/balance are used for trusted state.
- **Threat:** Save failures silently lose context needed for generation. **Mitigation:** Surface save failures and block generation only when required backend context cannot be created.
</threat_model>
