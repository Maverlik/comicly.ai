<threat_model>
- **Threat:** Client keeps decrementing credits locally instead of using backend balance. **Mitigation:** Update balance only from `/api/v1/me`, `/api/v1/wallet`, or generation response.
- **Threat:** Double-clicks double-generate or double-charge. **Mitigation:** Disable generation action and send `Idempotency-Key`.
- **Threat:** Edits during generation mutate in-flight request. **Mitigation:** Build immutable snapshot before async call.
- **Threat:** Provider base64 leaks into UI or storage assumptions. **Mitigation:** Use returned `image_url` only.
</threat_model>
