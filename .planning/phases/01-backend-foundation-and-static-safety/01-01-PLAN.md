<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| environment to app config | Untrusted or misconfigured env values enter app runtime. |
| HTTP client to FastAPI app | Untrusted requests enter app routes and validation handlers. |
| container build context to image | Local files may be copied into an image if `.dockerignore` is incomplete. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | I | `backend/app/core/config.py` | mitigate | Load only Phase 1 required config at startup and never include secret/env values in public errors. |
| T-01-02 | I | `backend/app/main.py` | mitigate | Do not mount `StaticFiles` or add file-serving routes, preserving API-only posture per D-23/D-24. |
| T-01-03 | T | `backend/app/core/errors.py` | mitigate | Normalize validation and application errors before business logic with stable machine-readable codes. |
| T-01-04 | I | `backend/.dockerignore` | mitigate | Exclude `.env`, dot directories, caches, planning artifacts, and backend docs from the Docker build context where possible. |
</threat_model>
